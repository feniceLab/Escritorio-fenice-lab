# Como Usar o Design System

Este guia mostra como integrar e usar o Starkën Design System em seus projetos.

## 📦 Instalação

### Via NPM (recomendado)

```bash
npm install @starken/design-system
```

### Via CDN

```html
<link rel="stylesheet" href="https://cdn.starkentecnologia.com.br/design-system/v2/starken-design-system.css">
```

### Download Manual

Baixe os arquivos direto do repositório e inclua no seu projeto:

```html
<link rel="stylesheet" href="./starken-design-system.css">
```

## 🚀 Configuração

### HTML Puro

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Meu Projeto</title>

    <!-- Importar fontes -->
    <link href="https://fonts.googleapis.com/css2?family=Poppins:wght@400;500;600;700&display=swap" rel="stylesheet">

    <!-- Design System CSS -->
    <link rel="stylesheet" href="starken-design-system.css">
</head>
<body>
    <div class="starken-container">
        <h1>Olá Mundo!</h1>
        <button class="starken-btn starken-btn-primary">Começar</button>
    </div>
</body>
</html>
```

### React

```jsx
// App.jsx
import '@starken/design-system/dist/starken-design-system.css';

function App() {
  return (
    <div className="starken-container">
      <h1>Olá Mundo!</h1>
      <button className="starken-btn starken-btn-primary">
        Começar
      </button>
    </div>
  );
}

export default App;
```

### Vue.js

```vue
<!-- App.vue -->
<template>
  <div class="starken-container">
    <h1>Olá Mundo!</h1>
    <button class="starken-btn starken-btn-primary">
      Começar
    </button>
  </div>
</template>

<script>
import '@starken/design-system/dist/starken-design-system.css';

export default {
  name: 'App'
}
</script>
```

### Next.js

```jsx
// pages/_app.js
import '@starken/design-system/dist/starken-design-system.css';

function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />
}

export default MyApp;
```

## 🎨 Usando Variáveis CSS

O design system fornece todas as cores, espaçamentos e outros tokens como variáveis CSS:

```css
.meu-componente-customizado {
  /* Cores */
  background: var(--starken-color-bg-surface);
  color: var(--starken-color-text-primary);

  /* Espaçamento */
  padding: var(--starken-spacing-4);
  margin-bottom: var(--starken-spacing-6);

  /* Bordas */
  border-radius: var(--starken-radius-md);
  border: 1px solid var(--starken-color-border-default);

  /* Transições */
  transition: all var(--starken-duration-normal) var(--starken-easing-default);
}

.meu-componente-customizado:hover {
  border-color: var(--starken-color-border-hover);
  box-shadow: var(--starken-shadow-glow);
}
```

## 📱 Responsividade

O design system é totalmente responsivo por padrão. Use o container para layouts responsivos:

```html
<div class="starken-container">
  <!-- Conteúdo com margens automáticas e responsive -->
</div>
```

Breakpoints disponíveis:
- **sm:** 640px
- **md:** 768px
- **lg:** 1024px
- **xl:** 1280px
- **2xl:** 1536px

### Boas práticas para uso desses assets

- Preserve a proporção original dos arquivos
- Use a logo branca em fundos escuros e a principal em superfícies neutras
- Evite aplicar efeitos extras ou alterar a paleta da marca
- Prefira os templates de post como base visual, mantendo consistência entre campanhas

## ♿ Acessibilidade

Todas as combinações de cores atendem WCAG 2.1 nível AA. Sempre:

✅ Use labels semânticos em formulários
✅ Forneça alt text em imagens
✅ Mantenha contraste adequado
✅ Use headings hierárquicos (H1 → H2 → H3)

```html
<!-- ✅ BOM -->
<label class="starken-label" for="email">E-mail</label>
<input type="email" id="email" class="starken-input" aria-required="true">

<!-- ❌ RUIM -->
<input type="email" class="starken-input" placeholder="E-mail">
```

## 🔧 Customização

### Sobrescrever variáveis

Você pode sobrescrever as variáveis CSS para customizar:

```css
:root {
  /* Alterar cor primária */
  --starken-color-brand-emerald: #your-color;

  /* Alterar espaçamento base */
  --starken-spacing-4: 1.25rem;

  /* Alterar fonte */
  --starken-font-family-primary: 'Sua Fonte', sans-serif;
}
```

### Criar componentes customizados

```css
.meu-card-especial {
  /* Usar tokens do design system */
  background: var(--starken-color-bg-surface);
  padding: var(--starken-spacing-6);
  border-radius: var(--starken-radius-lg);

  /* + suas customizações */
  border-left: 4px solid var(--starken-color-brand-emerald);
}
```

1. **Sempre use as classes do design system** ao invés de CSS inline
2. **Use variáveis CSS** para manter consistência
3. **Siga a hierarquia de headings** (H1 → H2 → H3)
4. **Teste em diferentes tamanhos de tela**
5. **Valide acessibilidade** com ferramentas como Lighthouse
